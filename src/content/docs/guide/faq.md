## 一般使用者 FAQ

### Q1：如何登入 Arova Nexus？

在瀏覽器輸入公司提供的 Nexus 網址，你會看到登入頁面。輸入帳號和密碼即可登入。如果公司已設定企業帳號登入（SSO），點選**企業帳號登入**按鈕，使用公司的 AD 帳號認證。

### Q2：忘記密碼怎麼辦？

在登入頁面點選**忘記密碼**，輸入你的 Email，系統會寄送密碼重設信件到你的信箱。點擊信件中的連結設定新密碼即可。如果沒收到信件，請聯繫 IT 管理員協助重設。

### Q3：Dashboard 上的數字代表什麼？

Dashboard 的統計卡片會顯示你的工單狀態摘要：未結工單數、待簽核數、本月提交數和已解決數。每張卡片有趨勢箭頭顯示跟上月的比較，點擊任何卡片可以跳到對應的列表頁查看詳情。詳見 [快速入門 — 認識 Dashboard](01-quick-start.md)。

### Q4：AI Copilot 可以幫我做什麼？

AI Copilot 是你的智慧助手。你可以用自然語言問它 IT 相關問題（例如「怎麼連 VPN」）、查詢工單狀態、提交服務申請。點選畫面右下角的 AI 懸浮按鈕即可開始對話。詳見 [快速入門 — 體驗 AI Copilot](01-quick-start.md)。

### Q5：如何調整通知設定？

點選右上角你的頭像，進入**個人設定** → **通知偏好**。你可以針對每種事件類型（工單狀態變更、簽核通知等）個別開關 Email 和站內通知。注意，部分關鍵事件（如 SEV1 Incident 通知）無法關閉。

### Q6：如何切換介面語言？

在頂部導航列找到語言切換按鈕，可以在**繁體中文**和 **English** 之間切換。切換會即時生效，系統會記住你的偏好。

### Q7：如何開啟深色模式？

在頂部導航列找到主題切換按鈕，選擇**深色**即可。也可以選**系統**，讓 Nexus 跟隨你的作業系統主題自動切換。偏好設定會被記住。

### Q8：遇到問題要怎麼求助？

你有三個管道：(1) 打開 AI Copilot 直接提問；(2) 在系統中提交服務請求，IT 團隊會收到通知；(3) 聯繫公司的 IT 管理員。如果是系統本身的問題，IT 管理員可以聯繫 Arova 技術團隊。

## IT 維運 FAQ

### Q1：Incident 的處理流程是什麼？

Incident 遵循固定的狀態流程：**觸發** → **已確認** → **調查中** → **處理中** → **已解決** → **已關閉**。收到新 Incident 後，先確認是否為真實問題（點選**確認**），再進行調查和修復。如果是誤報，可以直接關閉並填寫原因。詳見 [事件管理操作指南](02-incident-management.md)。

### Q2：AI 分析的準確度高嗎？

AI 的初步分析摘要基於警報內容、歷史事件和知識庫資料產生，準確度會隨著使用持續提升。每個 AI 建議都附有信心度分數供你參考。如果 AI 建議不準確，請透過回饋按鈕告訴系統，幫助 AI 改善。

### Q3：為什麼多個警報被合併成同一個 Incident？

Nexus 會自動將 5 分鐘內來源和類型相同的警報合併為同一個 Incident，避免重複建立。你可以在 Incident 詳情頁展開查看每筆原始警報。合併的警報數量會顯示在 Incident 卡片上。

### Q4：Incident 多久沒處理會被升級？

升級規則依嚴重度不同：SEV1 事件 15 分鐘內沒人確認，系統會自動通知 IT 管理員和全團隊；SEV2 事件 30 分鐘內沒人確認，會通知團隊主管。IT 管理員可以在 **設定** → **事件管理** → **升級規則** 自訂升級時間和通知對象。

### Q5：Workflow 什麼時候會被自動觸發？

Workflow 可以透過三種方式觸發：(1) 手動點選**執行**；(2) 排程自動觸發（依 Cron 設定）；(3) 外部系統透過 Webhook 或 API 觸發。觸發條件由 IT 管理員在流程設定中定義。詳見 [自動化操作指南](05-automation.md)。

### Q6：知識庫搜尋有什麼技巧？

你可以使用兩種搜尋模式：**語意搜尋**（用自然語言描述問題，例如「伺服器 CPU 過高怎麼處理」）和**關鍵字搜尋**（用精確關鍵字，例如法規編號或文件名稱）。使用知識庫選擇器可以限定搜尋範圍到特定的知識庫分區。切換到 **AI 問答**模式可以取得 AI 整理過的回答。詳見 [知識庫操作指南](04-knowledge-base.md)。

### Q7：如何設定排程報表？

到**分析報表**模組，選擇你要排程的報表，點選**排程**按鈕。設定執行頻率（每日、每週、每月）、收件人 Email 和報表格式。排程設定後會自動在指定時間產生報表並寄送。詳見 [分析報表操作指南](03-analytics-reporting.md)。

### Q8：Graylog / LibreNMS 警報沒有送到 Nexus 怎麼辦？

請依序檢查：(1) Graylog / LibreNMS 的 Webhook URL 是否正確；(2) API Key 是否有效且未過期；(3) 防火牆是否允許兩台伺服器之間的通訊。詳細的排查步驟請參考 [Graylog + LibreNMS 整合指南 — 問題排除](07-graylog-librenms-integration.md)。

## IT 管理員 FAQ

### Q1：如何新增使用者？

進入 **設定** → **使用者管理**，點選**新增使用者**。填寫 Email 和基本資訊後，使用者會收到啟用信件。你也可以在**待審核**頁籤中審核自行註冊的使用者。如果已設定 SSO（AD/LDAP/OIDC），使用者首次使用企業帳號登入時會自動建立帳號。

### Q2：如何設定角色權限？

進入 **設定** → **使用者管理** → **角色**。系統預設了 Service Desk Agent、IT Operations、Change Manager、Approver、Admin 五個角色。你可以新增自定義角色，並在權限矩陣中為每個角色設定各模組的查看/新增/編輯/刪除權限。一個使用者可以擁有多個角色，權限取聯集。

### Q3：如何設定簽核流程？

進入 **設定** → **簽核引擎**。你可以建立單級或多級串聯的簽核流程，為每一級指定簽核人（固定人員或依角色），並設定逾時時間。簽核流程可以被 Workflow 的簽核節點和服務請求呼叫。建立完成後，在需要簽核的 Workflow 中拖入簽核節點並選擇對應的流程即可。

### Q4：API Key 如何管理？

進入 **設定** → **API Key 管理**。你可以建立、查看和撤銷 API Key。建立時可以指定權限範圍（限定可存取哪些模組的 API）。每個 Key 會顯示最後使用時間，方便你追蹤使用情況。API Key 建立後只顯示一次，請立即複製保存。

### Q5：如何查看授權和使用量？

進入 **設定** → **授權管理**。頁面會顯示你目前的授權狀態：已啟用的模組（綠色）、未啟用的模組（灰色）、各項使用量進度條（使用者數、AI 呼叫次數、知識庫儲存空間等）和到期日。使用量超過 80% 會顯示黃色警示，超過 95% 會顯示紅色警示。

### Q6：AI 呼叫次數用完了怎麼辦？

AI 呼叫次數為月度配額，每月 1 日自動重置。如果配額用盡，AI Copilot 和知識庫 AI 問答會暫時停用，但其他功能不受影響。你可以聯繫 Arova 業務團隊升級配額，或等待下月自動重置。到 **設定** → **授權管理** 可以查看本月用量明細。

### Q7：如何匯出稽核日誌？

進入 **設定** → **稽核日誌**。使用篩選條件（操作類型、資源類型、使用者、日期範圍）縮小範圍後，點選**匯出 CSV**。稽核日誌預設保留 180 天（最少 90 天，可調整）。日誌內容不可修改或刪除。

### Q8：如何查看整合連線的健康狀態？

進入 **設定** → **整合管理**。頁面會列出所有已設定的外部系統連線（Graylog、LibreNMS、Email SMTP 等），每個連線會顯示狀態（已連線 / 未設定 / 錯誤）、最後成功時間和錯誤訊息。連線中斷時系統會自動發送通知給 IT 管理員。

## AI 專區 FAQ

### Q1：AI 可以做什麼？不能做什麼？

**AI 可以做的事：**
- 回答 IT 相關問題（基於知識庫內容）
- 分析 Incident 的可能原因和建議修復方案
- 自動產生事件摘要、工單摘要
- 協助觸發自動化流程（經你確認後才會執行）
- 自動分類和指派工單

**AI 不能做的事：**
- 直接修改系統設定或刪除資料（所有寫入操作都需要你確認）
- 回答知識庫中沒有的內容（AI 會明確告知「目前沒有相關資料」）
- 取代人工判斷（AI 提供建議，最終決策由你做出）

### Q2：為什麼 AI 每次的回答可能不同？

AI 使用語言模型產生回答，即使是相同問題，也可能因為上下文、知識庫更新或模型推理路徑不同而產生略有差異的回答。這是正常現象，不影響回答的正確性。如果你覺得某次回答特別好，可以點選**釘選**保存。

### Q3：如何對 AI 的回答給回饋？

每個 AI 回答的下方都有 👍（有幫助）和 👎（沒幫助）按鈕。點選 👎 時可以進一步填寫原因或提供修正。你的回饋會直接用於改善 AI 的回答品質，讓 AI 越用越準確。

### Q4：AI 無法使用時怎麼辦？

如果 AI 服務暫時不可用，系統會自動切換到經典模式。所有功能都能透過傳統 UI 操作完成，不會有功能缺失。Dashboard 也會從 AI 簡報自動退回靜態卡片顯示。AI 服務恢復後會自動切換回來。

### Q5：AI 呼叫有使用限制嗎？

有。AI 呼叫次數為月度配額，具體額度取決於你的授權方案。IT 管理員可以在 **設定** → **授權管理** 查看本月 AI 呼叫用量和剩餘額度。配額包含使用者發起的 AI 對話和 AI Agent 的背景呼叫。配額達 80% 時，AI Agent 會自動降低活動頻率以優先保留使用者端的 AI 額度。

### Q6：AI 會把我的資料送到外部嗎？

不會。Arova Nexus 的 AI 完全在你的本地環境（地端）運行，包括語言模型和向量資料庫。你的資料、對話內容和知識庫文件都不會離開你的伺服器。所有 AI 操作都會記錄在稽核日誌中。

## 問題排除指南

### 登入 / 認證

| 症狀 | 可能原因 | 解決方法 |
|------|---------|---------|
| 輸入正確密碼但無法登入 | 帳號被鎖定（連續失敗 5 次會鎖定 15 分鐘），或帳號尚未被 IT 管理員審核通過 | 等待 15 分鐘後重試，或聯繫 IT 管理員手動解鎖帳號、確認帳號審核狀態 |
| 點選「企業帳號登入」後跳轉失敗 | SSO 設定異常（AD/LDAP/OIDC 連線失敗） | 聯繫 IT 管理員檢查 SSO 連線設定；IT 管理員到 **設定** → **整合管理** 檢查 AD/LDAP 連線狀態 |
| 操作一段時間後被登出 | JWT Token 過期（Session 逾時） | 重新登入即可。如果頻繁發生，聯繫 IT 管理員調整 Session 逾時設定 |

### 整合

| 症狀 | 可能原因 | 解決方法 |
|------|---------|---------|
| Graylog / LibreNMS 警報沒有送到 Nexus | Webhook URL 設定錯誤、API Key 無效、防火牆阻擋 | 確認 URL 格式正確，重新檢查 API Key 是否有效，開放防火牆規則。詳見 [整合指南](07-graylog-librenms-integration.md) |
| Incident 的嚴重度與預期不符 | Payload Mapping 嚴重度對照表設定錯誤 | 到 **設定** → **整合管理** → **Monitoring Payload Mapping** 檢查並修正嚴重度對照規則 |
| 整合連線狀態顯示「錯誤」 | 外部系統連線中斷、認證資訊過期 | 到 **設定** → **整合管理** 點選該連線的**錯誤日誌**查看詳細錯誤訊息，依錯誤原因修正 |
| Email 通知沒有寄出 | SMTP 設定不正確或 SMTP 伺服器無法連線 | 到 **設定** → **整合管理** → **Email（SMTP）** 檢查連線設定，使用**發送測試信**功能驗證 |

### AI / Copilot

| 症狀 | 可能原因 | 解決方法 |
|------|---------|---------|
| AI Copilot 沒有回應 | AI 引擎服務異常，或本月 AI 呼叫配額已用盡 | IT 管理員到 **設定** → **系統健康** 檢查 AI 引擎狀態；到 **設定** → **授權管理** 檢查 AI 呼叫用量 |
| AI 回答與問題無關 | 知識庫中缺少相關資料，或問題描述過於模糊 | 嘗試用更具體的描述重新提問；IT 管理員確認相關文件已上傳至知識庫並處理完成（狀態為「就緒」） |
| AI 分析摘要沒有出現在 Incident 上 | AI 引擎繁忙或暫時不可用 | 等待 1-2 分鐘後重新整理頁面。如果持續未出現，IT 管理員到 **設定** → **系統健康** 檢查 AI 引擎 |

### 通知

| 症狀 | 可能原因 | 解決方法 |
|------|---------|---------|
| 沒有收到 Email 通知 | 個人通知偏好中 Email 被關閉，或 SMTP 整合設定異常 | 到 **個人設定** → **通知偏好** 確認 Email 通知已開啟；IT 管理員到 **設定** → **整合管理** → **Email（SMTP）** 檢查連線 |
| 站內通知數量紅點不消失 | 有未讀通知 | 點選頂部導航列的鈴鐺圖示，展開通知中心面板，點選**全部標為已讀** |
| 收到太多不重要的通知 | 通知偏好未依個人需求調整 | 到 **個人設定** → **通知偏好**，關閉不需要的事件類型通知。注意，部分關鍵事件通知（SEV1 Incident 等）無法關閉 |

### Workflow / 自動化

| 症狀 | 可能原因 | 解決方法 |
|------|---------|---------|
| Workflow 執行失敗 | 外部 API 連線失敗、腳本錯誤或逾時 | 到流程的**執行歷史**，點選失敗的執行紀錄查看各節點的詳細錯誤訊息，依錯誤原因修正 |
| 排程沒有在預期時間觸發 | 排程被暫停，或有效期間設定不正確 | 到 **自動化** → **排程管理** 確認排程狀態為「啟用」，檢查有效期間設定 |
| Workflow 卡在簽核節點不動 | 簽核人尚未處理，或簽核逾時設定過長 | 確認簽核人是否已收到通知。IT 管理員可以到 **設定** → **簽核引擎** 調整逾時提醒和自動退回時間 |
